# Git Sync

Sync all git repos in the current working directory across machines. Three modes: **audit**, **pull**, **push**.

**Script:** `~/.claude/skills/git-sync/scripts/git-sync.sh`

## Modes

### 1. `/git-sync audit` (default)

Run: `~/.claude/skills/git-sync/scripts/git-sync.sh audit <cwd>`

Parse structured output between `===REPO_START===` and `===REPO_END===` markers. Each repo block contains key=value lines: BRANCH, REMOTE_URL, FETCH, STAGED, UNSTAGED, UNTRACKED, AHEAD, BEHIND, DIVERGED, STASH, WORKTREE, REMOTE_ONLY, OPEN_PRS, plus TRACKING_START/END, UNTRACKED_FILES_START/END, REMOTE_ONLY_START/END, and OPEN_PRS_START/END blocks. Full format and field semantics: `~/.claude/skills/git-sync/references/output-format.md` — read it when parsing details matter (REMOTE_ONLY relationships, FETCH=SKIPPED, NOTE= lines).

**Present as a summary table:**

| Repo | Branch | Status | Ahead | Behind | Remote-only | Open PRs | Untracked | Action Needed |
|------|--------|--------|-------|--------|-------------|----------|-----------|---------------|

Status = WORKTREE value (CLEAN/DIRTY). Action Needed = derived from state.

**Priority ordering** (list repos needing attention below the table):
1. DIVERGED — warn prominently
2. DIRTY + BEHIND — commit or stash before pulling
3. DIRTY — uncommitted changes
4. BEHIND — needs pull
5. AHEAD — needs push
6. **REMOTE_ONLY with BASED_ON:<local-branch>** — "Someone has continued work on your branch"
7. **REMOTE_ONLY with AHEAD_OF_MAIN** — "New branches on the latest main"
8. **REMOTE_ONLY with BEHIND/DIVERGED_FROM_MAIN** — "Stale remote branches"
9. **OPEN_PRS > 0** — "X open PRs"
10. CLEAN — no action

**Detail section for Remote-only Branches** (below the main table, only for repos with REMOTE_ONLY > 0):

```
### <repo-name> — N remote-only branches

| Branch | Age | Relationship | Size | Last change |
|--------|-----|--------------|------|-------------|
| origin/feature-x | 2 days | BEHIND_MAIN (5 commits behind) | +12/-3 | Fix orientation bug |
| origin/dependabot/... | 1 day | AHEAD_OF_MAIN (current) | +8/-2 | Bump express 4.18->4.19 |

Assessment:
- origin/feature-x: Based on older main — check whether still relevant
- origin/dependabot/...: Current, builds on latest main — review/merge the PR
```

**Assessment logic** (generated by skill, not script):
- AHEAD_OF_MAIN → "Current, builds on latest main — relevant"
- BEHIND_MAIN (small) → "Slightly outdated, but probably still relevant"
- BEHIND_MAIN (large) → "Very outdated — likely stale, check whether to delete"
- DIVERGED_FROM_MAIN → "Parallel development — may need rebase/merge"
- BASED_ON:<local-branch> → "Builds on your branch — someone has continued work on it"
- SAME_AS_MAIN → "Identical to main — can be deleted"

**Open PRs section** (below remote-only, only for repos with OPEN_PRS > 0):

```
### <repo-name> — N open PRs

| PR | Title | Branch | Last updated |
|----|-------|--------|--------------|
| #42 | Fix orientation detection | claude/fix-orientation-detection | 2026-03-14 |
```

**Untracked file classification** (from UNTRACKED_FILES block):

| Pattern | Classification | Suggestion |
|---------|---------------|------------|
| `.pyc`, `__pycache__/`, `node_modules/`, `.next/`, `dist/`, `.build/` | Build artifact | Add to .gitignore |
| `.env*` | Environment file | Add to .gitignore, warn about secrets |
| `.ts`, `.py`, `.js`, `.md`, `.tsx`, `.jsx` | Source file | Suggest tracking |
| `.idea/`, `.vscode/`, `*.iml` | IDE file | Add to .gitignore |
| `.DS_Store`, `Thumbs.db` | OS file | Add to .gitignore |

**Also report:** stash count if > 0, `[gone]` branches from TRACKING block.

### 2. `/git-sync pull` (Start of Day)

1. Run `git-sync.sh fetch-all <cwd>` (fetches up to 8 repos in parallel)
2. Run `git-sync.sh audit <cwd> --no-fetch` (skips the per-repo fetch — fetch-all just did it)
3. For each repo where BEHIND > 0 and DIVERGED != YES:
   - Run `git-sync.sh pull <absolute-repo-path>`
4. For DIVERGED repos: warn user, use `AskUserQuestion` — options: skip / rebase / merge manually
5. For `[gone]` branches: offer cleanup via `git-sync.sh clean-merged <absolute-repo-path>`
6. For repos with REMOTE_ONLY > 0: show remote-only branches and ask if any should be checked out (`git checkout <branch-name>`)
7. Present summary: "X repos updated, Y already current, Z need attention, N remote-only branches found"

### 3. `/git-sync push` (End of Day)

1. Run `git-sync.sh audit <cwd>`
2. **BEFORE any commit:** classify untracked files (same table as audit) and resolve them —
   add build/IDE/OS artifacts and `.env*` to .gitignore (confirm with user), suggest tracking
   for source files. `git-sync.sh commit` stages with `add -A` and REFUSES if untracked
   `.env*` files are present — this step must come first.
3. For each DIRTY repo:
   a. Show change summary (STAGED/UNSTAGED/UNTRACKED counts)
   b. `AskUserQuestion` for commit message. Offer "auto" option → generates `"Update N files in <repo>"`
   c. Run `git-sync.sh commit <absolute-repo-path> "<message>"`
   d. Run `git-sync.sh push <absolute-repo-path>`
4. For AHEAD + CLEAN repos: run `git-sync.sh push <absolute-repo-path>`
5. For `[no upstream]` branches: ask if tracking should be set up via `git-sync.sh setup-tracking <repo-path> <branch>`
6. For repos with STASH > 0: inform user
7. Run final `git-sync.sh audit <cwd> --no-fetch` to confirm all synced
8. Present summary: "X committed, Y pushed, Z skipped"

## Parallelization

The audit output already contains everything needed (relationships, diffstats, PRs) —
no sub-agents required. Run independent pull/commit/push script calls as **parallel
Bash invocations in one message**. Keep sequential: anything needing user interaction
(commit messages, DIVERGED repos) and `gh`-heavy steps (API rate limits).

## Script Commands Reference

| Command | Arguments | Purpose |
|---------|-----------|---------|
| `audit` | `[dir] [--no-fetch]` | Scan all repos in directory |
| `fetch-all` | `[dir]` | Fetch all repos in directory (parallel, max 8) |
| `pull` | `<repo-path>` | Pull (ff-only) single repo |
| `push` | `<repo-path>` | Push single repo (never force) |
| `commit` | `<repo-path> <message>` | Stage all + commit |
| `setup-tracking` | `<repo-path> <branch>` | Set upstream tracking |
| `clean-merged` | `<repo-path>` | Delete local branches merged into main |

Action command output format: `===ACTION=== <cmd> <repo>` followed by `RESULT=OK|FAILED|SKIPPED` and `MESSAGE=...`

## Safety Rules

- **Never force-push** — the script only ever runs a plain `git push` (no flags forwarded)
- **Never commit untracked `.env*` files** — `commit` refuses; gitignore them first
- **Never auto-resolve conflicts** — always ask user
- **Always confirm commit messages** — unless user explicitly says "auto"
- **Network failure** — continue with next repo, report failures at end
- **DIVERGED repos** — warn prominently, do NOT attempt auto-merge

## Edge Cases

| Situation | Behavior |
|-----------|----------|
| Detached HEAD | BRANCH=DETACHED. Warn, suggest checkout main |
| No remote | REMOTE_URL=NONE, FETCH=SKIPPED. Skip sync, report as local-only |
| Network failure | FETCH=FAILED. Use local data, note in report |
| Untracked `.env*` in dirty repo | `commit` refuses. Gitignore first, then retry |
| cwd is itself a git repo | NOTE= line in audit output; only subdirectories scanned |
| Merge conflicts | Unmerged paths detected. Warn, do NOT auto-resolve |
| Pull not ff-able | DIVERGED. Offer rebase/merge via AskUserQuestion |
| `[gone]` branches | Offer cleanup via clean-merged |
| Empty repo | Report as found, note empty state |
| `gh` CLI not installed | OPEN_PRS=UNAVAILABLE. Skip PR info, note in report |
| `gh` API timeout | OPEN_PRS=UNAVAILABLE. Graceful fallback |

## Output Parsing

Full audit/action output format and REMOTE_ONLY/OPEN_PRS field semantics:
`~/.claude/skills/git-sync/references/output-format.md`

Key points: relationships are computed against the **remote** default branch
(`origin/HEAD`), so they stay correct even when local main is stale.
`FETCH=SKIPPED` = no remote or `--no-fetch`. A `NOTE=` line appears when the
target directory is itself a git repo (it is not audited, only subdirectories).
